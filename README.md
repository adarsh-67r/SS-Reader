## Quick Start

### 1. Add Your EPUB

Place your `Shadow Slave.epub` in the **root of the repository**:

```bash
cp ~/Downloads/shadow-slave.epub "Shadow Slave.epub"
git add "Shadow Slave.epub"
git commit -m "Add Shadow Slave EPUB"
git push
```

GitHub Actions will automatically process the EPUB and deploy the site!

### 2. Local Development

```bash
# Install root dependencies (for EPUB processing)
npm install

# Process EPUB to extract chapters
npm run process-epub

# Validate chapter formatting
npm run validate

# Start development server
cd website
npm install
npm run dev
```

Visit http://localhost:5173

### 3. Build for Production

```bash
cd website
npm run build
# Output in website/build/
```

## EPUB Processing

The `scripts/process-epub.js` script:
1. Reads `Shadow Slave.epub` from the repo root
2. Parses the EPUB spine (chapter order)
3. Converts each chapter's HTML to Markdown
4. Writes chapter files to `chapters/*.md` with YAML frontmatter
5. Generates `chapters/metadata.json`

```bash
node scripts/process-epub.js
```

### Regenerate Index Only

If you've manually edited chapter files and need to rebuild `metadata.json`:

```bash
node scripts/generate-index.js
```

### Validate Formatting

Check all chapters for formatting issues:

```bash
node scripts/validate-format.js
```

## Configuration

### Giscus Comments

To configure comments, update the `<Giscus>` component in
`website/src/routes/chapter/[slug]/+page.svelte` with your own
`repo`, `repoId`, `category`, and `categoryId` values from
[giscus.app](https://giscus.app).

### Base Path

If your GitHub Pages URL has a subpath (e.g., `/SS-Reader`), it is
automatically set from the repository name via `BASE_PATH` in the
deploy workflow.
